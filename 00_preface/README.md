# 前言：从一把铁锤到一张格栅

---

### 我身边到处都是拿着锤子的人

做网络工程师的那些年，我身边到处都是「手拿锤子的人」。

遇到业务问题——上设备。遇到流程问题——上系统。遇到管理问题——上制度。每个人都用自己的锤子敲打世界，而且敲得理直气壮。

我不是例外。我从自己的锤子里看出去，答案总是一样的：网络架构不够健壮。但我隐约察觉到不对劲——同一个问题，换一个部门来看，解法完全不一样，而每个部门都坚信自己才是对的。技术部门推系统，运营部门推流程，管理部门推考核。每个人都在用锤子敲，只是锤子不同。

我说不清楚到底哪里不对。直到有一天，一个人替我说了出来。

### 《穷查理宝典》给我的那一巴掌

我翻开查理·芒格的《穷查理宝典》，看到那句让我头皮发麻的话：

> 手里只有铁锤的人，看什么都像钉子。

原来不是我有问题——是所有人的工具箱里都只有一把锤子，包括我自己。

芒格在那本书里提出了「多元思维模型」和「格栅理论」：真正的智慧不在于精通单一学科的深度，而在于搭建多学科的格栅框架——你不必成为每门学科的专家，但你要掌握每个学科最核心的那几个大想法。如果你只懂经济学，你会用供求曲线解释婚姻；如果你只懂心理学，你会用童年创伤解释经济危机；如果你只懂工程学，你会用系统冗余解释一切人际摩擦。你对，但你只对了一部分。

芒格指出了方向。但他没有画地图。

他告诉你格栅思维很重要，却没有告诉你怎么搭——哪些学科值得了解？每个学科最核心的模型是什么？模型与模型之间怎么连结？如何在日常决策中真正用起来？

这本书就是来画这张地图的。

### 为什么我要写这本书

写这本书的念头，最初来自一个很私人的需求：我想给自己的格栅搭一个框架。

从网络工程师转型做产品经理，从技术思维切换到商业思维，我深刻体会到单一框架的局限。技术背景让我看到系统，但看不到人；做产品让我看到需求，但看不到成本；做投资让我看到概率，但看不到人性。每一种视角都是真的，但每一种都是局部的。

我花了大量时间翻阅不同学科的入门书籍，寻找每个学科最核心的思维模型。物理学的、数学的、心理学的、经济学的……越读越觉得，这些模型之间有着惊人的内在联系——熵增定律可以解释组织僵化，复利效应可以贯穿财富增长和知识积累，认知误判可以预警投资陷阱。但现有的书要么只讲单一学科，要么只是模型词典——罗列100个模型，彼此毫无连结。

芒格说格栅的力量在于结与结之间的连线，而我能找到的书，只有结，没有线。

于是我决定自己来——把芒格的格栅理论从一个概念，变成一套可操作系统。从14个学科中提取100多个思维模型，标注13个核心模型深入展开，画出模型之间的连结图谱。不是百科全书，是连结地图。不是让你记住每个模型的定义，而是让你看到模型与模型之间的隐秘通道。

### 对结果的理解

写完前五章之后，我对格栅思维的理解比动笔前深了一层。有三个认知值得在这里说清楚：

**第一，懂一个学科和懂十几个学科的差别，远超想象。** 不是多知道几个名词那么简单——是多出一整套看待问题的角度。当你同时用熵增定律、认知误判和供给需求来看同一个现象时，你看到的东西和只懂一个学科的人截然不同。

**第二，模型之间的连结比模型本身更重要。** 100多个模型不是100多个孤立的知识点，而是一张网。熵增→耗散结构→反脆弱→冗余备份，这条链条一旦建立，你看组织、看投资、看人生的视角都变了。连结才是格栅的精髓。

**第三，不需要精通每个学科。** 掌握每个学科最核心的几个大想法就够了。芒格说的「80-90个重要模型就能让你拥有普世智慧」，其实真正关键的更少——本书标注的核心模型一共13个，吃透它们，格栅的骨架就立起来了。

这不是芒格替我们写的书。芒格给了方向，我动手搭了格栅。

## 写在前面

这本书写给所有用单一框架思考却不自知的人。

不是因为你愚蠢，而是因为我们的教育体系就是这样设计的——学物理的不学心理学，学经济的不学生物学，学管理的把一切简化为框架和矩阵。你用锤子敲了那么多年，不是因为锤子好，而是因为你工具箱里只有锤子。

你可能已经隐约意识到了：同样的问题，工程师看到系统冗余，心理学家看到认知偏差，投资人看到安全边际——每个人都对，但每个人都只看到一部分。

**这本书的目标是帮你搭建一把跨学科的格栅——不是让你成为每门学科的专家，而是让你在面对复杂问题时，能同时调用多个学科的视角，看到别人看不到的全貌。**

你不需要精通量子力学。你不需要背数学公式。你甚至不需要读过《穷查理宝典》——虽然读过会让某些章节更亲切。

但动手之前，有三个关键认知需要建立。

**第一，每个学科都是摸象的瞎子。** 物理学家看见世界的物质规律，心理学家看见人的认知偏差，经济学家看见资源配置的逻辑。每一门学科都有独特视角，也有盲区。如果你只懂一门，你就只能摸到大象的一条腿。

**第二，连结比知识更有力。** 知道「熵增定律」的定义不算什么，知道「熵增定律→组织衰老→反脆弱→冗余备份」这条链条才算什么。格栅的力量来自结与结之间的连线，而不是结本身。

**第三，思维模型的终极目标是从系统二走向系统一。** 不是每次决策都要做冗长的多模型分析——那太累了。目标是通过系统二的反复训练，将模型内化为系统一的直觉，像老司机开车一样自动运行。
