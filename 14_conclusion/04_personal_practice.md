# 作者个人实践总结


写了这么多模型，讲了这么多理论，如果不把自己也放上去验证，那就是纸上谈兵。这一节，我把自己的三次实践放在格栅理论的显微镜下——不是为了证明模型有多好用，而是为了诚实地展示：模型在真实的场景中是如何起作用的，以及在哪些地方它们也不够用。

## AWS SAP备考：7个模型的排列组合

2023年，我决定备考 AWS Solutions Architect Professional（SAP）认证——这是 AWS 架构师系列中难度最高的考试，官方建议至少2年实战经验。

作为一个非科班的电信工程师，我的起点并不高。但100天的备考，我用了一套系统的模型组合，最终以822/1000的分数通过（及格线750）。

回头看，这次备考是7个模型的Lollapalooza效应：

**排列组合（MT-2）**：AWS有200+服务，如果没有系统化的归类方式，你会在信息的海洋中溺水。我把所有服务按「计算-存储-网络-安全-数据库-消息-监控」七大门类排列组合，每个门类内再按「核心服务-辅助服务-边缘服务」分级。排列组合不是简单的分类——它帮你发现哪些服务是必考的高概率考点，哪些是低概率的边缘知识。结果是：我用20%的时间覆盖了80%的考点。

**费曼学习法**：每个 AWS 服务，我学完之后的检验标准不是「我看了教材」，而是「我能用大白话给同事讲清楚这个服务是干嘛的，什么场景用，什么场景不用」。讲不出的，回去重新学。费曼学习法强迫你跳出「我好像懂了」的错觉，真正检验你是否把信息转化为了知识。

**艾宾浩斯遗忘曲线**：按照1天、3天、7天、14天、30天的间隔重复安排复习。第1天看完的知识，第2天快速回顾一遍；第7天再做一组相关练习题；第30天做模拟考试的整体回顾。间隔重复的效果远超集中突击——同一个知识点，间隔重复5次的总时间不到集中突击的一半，但记忆保持率高出数倍。

**临界点（PC-6）**：备考的前60天，我做模拟题的正确率一直在55%-60%徘徊，感觉怎么学都没进步。这是典型的临界点前的平台期。如果我不了解临界点模型，我可能会在这个阶段怀疑自己的方法，甚至放弃。但我知道临界点的存在——在量变积累到临界值之前，质变不会发生。于是我坚持原来的方法，不加量、不改方向。第65天开始，正确率突然跳到75%，并且在随后的两周稳步爬升到85%。临界点的跨越不是渐进的，是突变的——而你需要做的，只是在临界前不要放弃。

**刻意练习**：刷题不是目的，获取反馈才是。每道错题我记录三个信息：(1) 错误类型（知识盲区/理解偏差/审题失误），(2) 涉及的 AWS 服务，(3) 正确的思考路径。每周回头看错题类型分布，发现「安全场景的设计题」错误率最高，就把下一周70%的时间放在安全服务的深度理解上。这种有目标、有反馈、有聚焦的练习，效率远超无差别刷题。

**元认知**：每周日晚，我用15分钟问自己三个问题：(1) 本周的学习方法有没有需要调整的？(2) 我是否在舒适区里浪费时间？（比如反复看已经掌握的内容）(3) 离考试还有X天，我的进度是否在正轨上？有一次元认知检查发现，我在 VPC 网络上花了过多时间（已经80%正确率了），而数据库服务的正确率只有50%。我立刻调整了时间分配——这就是元认知的价值：它不是让你学得更多，而是让你学得更对。

**杠铃策略**：80%的时间放在核心服务（EC2、S3、VPC、IAM、RDS、Lambda、CloudFormation）的深度理解上，20%的时间花在200+边缘服务的存在性认知上（只需要知道「有这个服务」和「大致干什么的」）。核心服务考得深，边缘服务考得浅——杠铃策略让我把有限的认知资源分配在了边际收益最大的地方。

7个模型，不是7个独立的工具，而是7根绳子拧成的一股缆：排列组合搭建知识框架，费曼学习法确保理解的深度，艾宾浩斯保证记忆的持久，临界点给你熬过平台期的信念，刻意练习提供有效的训练方法，元认知保证方向的正确，杠铃策略优化资源的分配。**822分不是因为我聪明，是因为7个模型的Lollapalooza效应。**

## 从电信工程师到产品经理：4个模型的转型决策

几年前，我面临一个职业选择：继续做电信工程师，还是转型做产品经理？

这个决定不是拍脑袋做的。我用了4个模型来支撑决策。

**机会成本（EC-1）**：留下的代价是什么？电信行业的增长在放缓，技术栈在老化，职业天花板清晰可见。如果继续留下，我付出的不是时间——时间反正要花——而是机会：我放弃了在互联网行业高速增长期切入的可能性。机会成本让我看清：不选择也是一种选择，而「不变」的代价可能比「变」的代价更大。

**费马帕斯卡（MT-4）**：转型的期望值是多少？概率端：我有技术背景（通信工程+编程能力），有对产品的热情（一直在业余时间研究产品），但缺乏产品经理的直接经验和行业人脉——综合估算转型成功的概率约60%，不算高。赔率端：如果成功，收入增长空间2-3倍，职业发展空间从线性变为指数型；如果失败，最差情况回到工程师岗位，损失1-2年时间。期望值 = 0.6 × (2.5倍回报) - 0.4 × (1年沉没成本) = 正值。期望值为正，值得下注。

**刻意练习**：转型不是换个头衔，而是换一种思维方式和技能体系。我列出了产品经理的核心能力地图（需求分析、用户研究、数据驱动、跨部门协作、商业思维），对照自己的现状找差距，然后针对每个差距设计刻意练习的方案。比如「用户研究」是最大短板，我就在转型前3个月每天花1小时读用户研究案例，每周末找一个真实产品做用户访谈练习。

**能力圈（IN-2）**：转型不是跳到自己完全不懂的领域，而是从能力圈的边缘向外扩展。我的技术背景是能力圈内的核心资产——我比纯商科背景的产品经理更懂技术可行性，这种差异化能力是我转型的竞争优势。我不去投纯运营类产品岗（离能力圈太远），而是投技术型产品岗（在能力圈边缘，向外扩展）。

四个模型从不同角度照亮了同一个决策：机会成本告诉我「不变有代价」，费马帕斯卡告诉我「期望值为正值得做」，刻意练习告诉我「怎么缩小差距」，能力圈告诉我「从哪里切入最有效」。当四个模型同时指向「转型」，我就有了行动的信心——不是盲目的勇气，而是有分析支撑的决断。

## 港股投资实践：5个模型的实盘验证

过去三年，我在港股市场进行了一系列投资实践。这是思维模型在最高压环境中的检验——你读了一百个模型，面对屏幕上跳动的数字和账户上缩水的资产，还能不能保持理性和纪律？

**能力圈（IN-2）**：我只买自己真正懂的行业——电信、云计算、互联网平台。港股有很多便宜的地产股和金融股，但我不懂房地产行业的周期逻辑，也不懂银行的不良资产判断。看不懂不买，这个纪律帮我避开了2022年港股地产股的集体崩盘。

**护城河（IN-3）**：在我懂的行业里，我只买有明确护城河的公司。腾讯的网络效应（用户越多，社交粘性越强），中国移动的规模效应+特许经营（基础设施行业天然垄断），阿里云的飞轮效应（更多客户→更多数据→更好的产品→更多客户）。这些公司的共同特征是：护城河不仅存在，而且在变宽，而不是变窄。

**市场先生（IN-5）**：2022年10月，港股恒生指数跌到14500点，市场先生处于极度恐慌状态——每天都是坏消息，每天都有人在割肉。但正是这个时候，我关注的公司估值已经到了历史最低区间。市场先生的恐慌给了我用远低于内在价值的价格买入的机会。我在这个时间段分批建仓，不是因为我很勇敢，而是因为我用能力圈和护城河确认了公司的质量，用安全边际确认了价格的保护，市场先生的恐慌只是给了时间窗口。

**安全边际（BU-1）**：每一笔买入，我都要求至少30%的安全边际——我估算的内在价值比买入价至少高30%。这意味着即使我的估值有30%的误差（几乎是一定会有的），我仍然不会大亏。安全边际在2022年的极端行情中证明了它的价值：我买入的标的在后续的进一步下跌中最多又跌了15%，但因为安全边际足够，我没有被迫割肉，最终在2023年的反弹中全部回正。

**格栅理论本身**：以上四个模型不是逐个调用的，而是同时作用的。每一次买入决策，我都在问：这家公司在我的能力圈内吗？（是）护城河在变宽还是变窄？（在变宽）市场先生是恐慌还是贪婪？（恐慌）安全边际够吗？（30%以上）当四个答案同时指向「买」，我才下单。格栅理论在这里不是第114个思维模型，而是所有模型的元模型——它要求你用多个维度交叉验证，在信号共振时行动，在信号冲突时等待。

实盘的结果不是这段话的重点——重点是：**每一次我偏离了模型纪律，结果都不好；每一次我严格遵守了模型纪律，即使短期看起来「亏了」，长期结果都验证了决策的正确性。** 2023年初有一只股票，我在安全边际只有15%的时候「觉得应该够了」就买入了，结果后续继续下跌20%，被迫在更低的位置加仓摊薄——如果当初严格遵守30%的安全边际，这次加仓根本不需要发生。纪律不以感觉为准，以模型为准。

## 思维模型不是万能药

说完了三个成功案例，我必须说一个更重要的事实：**思维模型不是万能药。**

它不能保证你每次都做对——概率论告诉我们，即使你用对了模型、做对了分析，单次决策仍然可能出错，因为现实世界有太多的随机性和不可控因素。

它不能替代专业知识——思维模型给你的是思维框架，不是具体知识。你用护城河模型选股，如果你不懂行业的具体运作机制，你依然判断不出护城河是在变宽还是变窄。模型是刀，知识是磨刀石——刀再锋利，也需要磨刀石不断地磨。

它不能解决所有问题——有些问题本质上是价值观问题（该不该做），不是方法论问题（怎么做）。第一性原理可以帮你理清事实，但不能替你做价值判断。

但它能做一件事——**提高你做对的概率。**

而概率，是这个不确定世界中唯一确定的东西。
